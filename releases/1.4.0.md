### 1. BEFORE UPGRADING (!!! IMPORTANT !!!)

1. Update the maintenance page with appropriate dates and details. Maintenance page is found at `/sw_ux/httpd0(1|2)/www/html/maintenance.html`
2. Put system into maintenance mode:
```shell
touch /apps_ux/apps_(dlvr|test|prod)/maintenance/bcrhp
```
3. Confirm system in maintenance mode by accessing:
`https://(|dlvr|test)apps.nrs.gov.bc.ca/bcrhp`. System should not be live, and maintenance page should be displayed.
4. Backup database:
```shell
sudo -u wwwadm /bin/bash
cd /apps_ux/apps_[env]/backups/database
chmod 700 .
pg_dump -Upostgres bcrhp_[env] > 2026-03-03_bcrhp_pre_1.4.0.sql
gzip 2026-03-03_bcrhp_pre_1.4.0.sql
chmod 400 2026-03-03_bcrhp_pre_1.4.0.sql.gz
chmod 500 .
```

5. Backup venv
   `mv bcrhp_<env>_venv backups/bcrhp_<env>_venv.2026-03-03`

### 2. Rollback patched migration
1. `psql -Upostgres bcrhp_<env> -c "delete from django_migrations where name = '12008_alter_file_path'"`


### 3. Perform Application deployment
The source code branches are as follows in the `cd/docker/config/app_env` file:
```yaml
FORCE_CORE_PIP_INSTALL=True

CORE_REPO_TAG=v8.0.10-bcgov
COMMON_REPO_TAG=v2.0.0
CONFIG_BRANCH=v1.4.0
```

Ensure the ansible source branch is set properly in the `cd/docker/run_deployment.sh` file:
```shell
export ANSIBLE_BRANCH=main
```

The migration variables must be set as follows (empty line ensure migrations are complete):
```yaml
perform_migration: False
```

```bash
docker start bcrhp-deployer
cd/docker/run_deployment.sh <dlvr | test | prod>
```

### 4. Deploy new version of application
```shell
sudo -uwwwadm /bin/bash
cd /apps_ux/apps_<env>
source bcrhp_<env>_venv/bin/activate
cd bcrhp
python manage.py migrate models
python manage.py migrate
python manage.py graph publish --update -ui
python manage.py check --deploy --settings=bcrhp.settings
python manage.py validate
python manage.py validate --fix 1012
python manage.py bc_reindex_database
exit
 ```

NB - BCGov Ansible deployments rebuild the front end assets automatically. This needs to be done manually from within the container when upgrading a local Docker instance:
```shell
rm -rf node_modules package-lock.json && npm install
npm run build_development
```

### 5. Restart the apache server
```shell
sudo systemctl restart httpd0(1|2)
```

### 6. Restart the celery broker
The celery broker needs to be restarted to pickup the source code changes.
```shell
sudo -uwwwsvr /bin/bash
s6-svc -d /apps_ux/s6_services/bcrhp_celery_<env>/
s6-svc -u /apps_ux/s6_services/bcrhp_celery_<env>/
exit
```
